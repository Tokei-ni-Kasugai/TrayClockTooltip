# 変更履歴

## 1.2.1 - 2026-06-15

- フローティング時計の時刻表示を、Windowsの短い時刻形式に従うようにしました。秒がない設定では秒を補完表示します。
- アプリ起動中にWindowsの日付/時刻形式を変更した場合、フローティング時計の幅と位置を再計算するようにしました。
- 画面端付近でのリサイズやドラッグ時に、フローティング時計がモニターの作業領域内に収まるようにしました。
- 固定表示中のフローティング時計が、トレイメニューを開いたときに非表示にならないようにしました。
- 短い時刻形式と固定表示中のトレイメニュー挙動に合わせて、テストケースとドキュメントを更新しました。

## 1.2.0 - 2026-06-14

- トレイメニューに通知しきい値の選択を追加しました。`1 sec`、`3 sec`、`5 sec`、`7 sec`、`10 sec`、`Off` から選択できます。
- 通知しきい値は、ずれ通知と通常メニューでの `Adjust Windows time (admin)` 表示に使用します。アプリ内時計は従来どおり取得したNTP時刻を使用します。
- しきい値変更は最新の既知NTPオフセットへ即時反映し、Startup登録がある場合は登録済みEXEパスを維持したまま起動引数だけを更新します。
- 通知本文は、日本語のWindows UI環境では日本語、日本語以外の環境では英語で表示するようにしました。
- 未インストールかつ自動起動未登録のポータブル起動で、ずれ通知が不要な場合に起動確認通知を表示するようにしました。
- インストール引き継ぎ後、インストール先EXEからインストール成功通知を表示するようにしました。
- インストール時、起動元EXEの隣に配布用READMEが存在する場合はインストール先フォルダへコピーするようにしました。
- 通知しきい値に合わせてテストケースとドキュメントを更新しました。

## 1.1.2 - 2026-06-12

- インストール後に起動したインストール先EXEが、ポータブル起動元フォルダをカレントディレクトリとして保持しないように修正しました。
- インストール引き継ぎ時の終了要求処理を調整し、調査用にインストール処理の段階ログを追加しました。
- インストール処理の段階ログは、ポータブル起動元フォルダ削除後も確認できるようにユーザーログフォルダへ記録します。

## 1.1.1 - 2026-06-11

- `Open log folder` の実行方式を、短時間だけ起動するヘルパープロセス経由に変更しました。
- Explorer 本来のフォルダ再利用動作を維持しつつ、常駐アプリ側に残るメモリを抑えるようにしました。

## 1.1.0 - 2026-06-11

- トレイメニューから、現在のユーザー向けの配置と自動起動登録を行えるようにしました。
- 別の場所にあるEXEから、インストール済みEXEを置き換えられるようにしました。
- NTP取得と時刻調整の履歴ログを追加し、256KBでローテートするようにしました。
- `Shift + 右クリック` メニューから `Open log folder` を開けるようにしました。
- 既知のずれが1秒未満の場合でも、`Shift + 右クリック` から時刻調整を呼び出せるようにしました。
- セットアップとログ機能に合わせて、GitHub Pagesとドキュメントを更新しました。

## 1.0.0 - 2026-06-05

初回リリース。

- 軽量なネイティブWin32タスクトレイ時計。
- 標準ツールチップの代わりに、独自のフローティング時計を表示。
- Windows Timeに設定されたNTPサーバを利用した、NTP基準のアプリ内時刻。
- 起動時、ログオン時、ロック解除時のNTP取得。
- トレイメニューからの手動NTP再取得。
- Windows時刻とNTP時刻が1秒以上ずれている場合の通知。
- UAC確認とNTP再取得を伴う時刻調整。
- Windowsモードに追従し、5秒で閉じる独自通知。
- Windowsモードに追従するフローティング時計。
- 固定表示中のフローティング時計にサイドインジケータを表示。
- Windowsのユーザー日付/時刻形式に追従し、秒がない設定では秒を補完。
